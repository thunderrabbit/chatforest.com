**At a glance:** 15+ servers. Workflow frameworks, multi-agent swarms, task management, gateway routing, protocol bridges.

## Agent Frameworks

**[mcp-agent](https://github.com/lastmile-ai/mcp-agent)** (8.1K stars) — composable Anthropic patterns: parallel, orchestrator-worker, evaluator-optimizer, routers, map-reduce. Full MCP support. Multi-provider LLM. Temporal-backed production execution.

**[fast-agent](https://github.com/evalstate/fast-agent)** (3.7K stars) — chain, parallel, evaluator-optimizer, router, agents-as-tools, MAKER K-voting. Anthropic/OpenAI/Google native. OAuth v2.1.

**[Agent-MCP](https://github.com/rinadelph/Agent-MCP)** (1.2K stars) — multi-agent parallel execution with persistent knowledge graph. Functions as an MCP server for Claude Desktop/Cline.

## Swarm Orchestration

**[Ruflo](https://github.com/ruvnet/ruflo)** (21.1K stars) — 60+ agents, 215 MCP tools, self-learning memory, fault-tolerant consensus. v3.5.0 production-ready (Feb 2026).

**[cli-agent-orchestrator](https://github.com/awslabs/cli-agent-orchestrator)** (AWS Labs) — hierarchical agents in isolated tmux sessions. Supervisor→worker delegation.

## Task Orchestration

**[task-orchestrator](https://github.com/jpicklyk/task-orchestrator)** (170 stars, Kotlin) — persistent work item graph, quality gates, phase-based progression, 13 MCP tools.

## Gateway & Protocol Bridges

**[ultimate_mcp_server](https://github.com/Dicklesworthstone/ultimate_mcp_server)** (143 stars) — multi-provider LLM, browser automation, memory, RAG. **[A2A-MCP-Server](https://github.com/GongRzhe/A2A-MCP-Server)** (145 stars) — bridges MCP↔Google A2A protocol.

## What's Good

Production-ready frameworks (mcp-agent, fast-agent). Genuine multi-agent innovation. Enterprise infrastructure emerging.

## What's Not

No agent discovery in MCP. No cost-aware scheduling. Limited observability across orchestrated agents.

## Rating: 4.0/5

Strong frameworks, innovative patterns, growing infrastructure. Top-tier projects are solid and actively maintained.
